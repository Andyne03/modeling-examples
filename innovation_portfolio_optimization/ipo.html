Motivation¶

Project and Innovation Portfolio Optimization is a problem that is prevalent in many industries. For example:

  • Major global IT organizations have hundreds of IT projects impacting hundreds of applications. Typically, these global IT organizations have budgets that amount to billions of dollars.
  • Major consulting and service providers need to manage a complex pipeline of clients’ project opportunities.
  • Fortune 500 global companies need to manage a complex funnel of technology innovation opportunities.
  • U.S county, state, and federal government organizations need to manage billions of dollars in projects that benefit U.S tax payers.

Portfolio management requires removing operational inefficiencies to generate savings that can be invested to fuel innovation. But optimizing these investments is not just about cutting costs, it is also about making investment decisions that deliver value to the business. Portfolio management involves frequently determining where to trim, where to invest, and where to shift to innovation. That is, portfolio management requires an efficient and repeatable approach so that companies can rapidly adapt to changes in business requirements and the shifts of the technology landscape.

Project and Innovation Portfolio Optimization is an extremely difficult problem to solve. There are an astronomical number of possible combinations from which to select, and companies must be able to schedule projects optimally with limited available resources. In addition, there are several conflicting business objectives that must be considered when creating a portfolio; consequently, there is a need to optimize the trade-offs between these conflicting objectives. Using traditional, manual tools and techniques, the “optimization” of a portfolio is a very time-consuming process that typically produces sub-optimal results and leads to delays in projects and processes.

Problem Description¶

How to optimize the selection and scheduling of a portfolio of projects so that the trade-offs among various conflicting business objectives are optimized, while satisfying resource constraints (e.g. labor availability and budgets) and other business constraints (e.g. project precedence constraints).

Model Formulation¶

Mathematical optimization or mathematical programming is a declarative approach where the modeler formulates a mathematical optimization problem that captures the key features of a complex decision problem. The Gurobi Optimizer solves the mathematical optimization problem using state-of-the-art mathematics and computer science.

A mathematical optimization model has five components:

  • Sets.
  • Parameters.
  • Decision variables.
  • Constraints.
  • Objective function(s).

MIP Model Description¶

  • The goal of this model is to optimize the selection and scheduling of a set of projects.
  • For each project, there is a benefit value and a customer satisfaction score.
  • The primary objective is to maximize the total benefit generated by the selected projects.
  • The secondary objective is to maximize the total customer satisfaction score generated by the selected projects. There is a threshold parameter (percentage) that allows degrading the optimal primary objective value in order to maximize the secondary objective while ensuring that at least the decreased value of the primary objective is achieved. To rank the impact of a project on customer satisfaction, the following scale can be used:
    • 100 means very high
    • 75 means high
    • 50 means average
    • 25 means low
    • 0 means none
  • For each project, there is a cost of selecting each project and a duration. Additionally, each project has a release period that indicates the earliest period in which a project can start, and a due period that defines the latest period in which the project can be completed.
  • There is a constraint that establishes that the costs of the selected projects cannot exceed a global budget.
  • There is a constraint that forces some projects to be selected.
  • There is a constraint that forces some projects not to be selected.
  • The value of the benefit and customer satisfaction that a project generates decreases over time.
  • The constraint that forces some projects to be selected may violate the budget constraint. Hence an artificial variable is added to the budget to correct the budget to fund the projects that are forced in.

Sets and Indices¶

$p \in P$: index and set of projects.

$IN \subseteq P$: set of projects forced to be selected.

$OUT \subseteq P$: set of projects forced to be not selected.

Parameters¶

$\alpha$: a discount factor to decrease the value of a project benefit and customer satisfaction as a function of time.

$BigM$: large penalty number for exceeding one unit of budget.

$\beta$: budget to fund the selected projects.

$\lambda_p$: release date of project $p \in P$

$\delta_p$: due date of project $p \in P$

$v_p$: value or benefit of project $p \in P$

$c_p$: cost of project $p \in P$

$d_p$: duration of project $p \in P$

${s_p}$: customer satisfaction of project $p \in P$

Computed Parameters¶

$[ES_p, LS_p]$: defines the earliest start and latest start time window of project $p \in P$. This time window is calculated as follows.

  • $ES_p = \lambda_p$
  • $LS_p = \delta_p - d_p$.

$[EF_p, LF_p]$: defines the earliest finish and latest finish time window of project $p \in P$. This time window is calculated as follows.

  • $EF_p = \lambda_p + d_p$
  • $LF_p = \delta_p$.

Decision Variables¶

$y_p = 1$ if project $p$ is selected in the portfolio; and 0 otherwise.

$x_{p,t} = 1$ if project $p$ starts at time period $t$; and 0 otherwise.

$z_{p,t} = 1$ if project $p$ is completed at time period $t$; and 0 otherwise

$ArtBudget$: takes a positive value in case the recommended budget does not cover the cost of all the selected projects.

MIP Model Formulation¶

Primary objective: Maximize the total discounted benefit generated by the selected projects.

$$ MAX \; Benefit = \sum_{p \in P} \sum_{t = EF_p}^{LF_p} \; \frac{v_p\;z_{p,t}}{(1 + \alpha)^t} - BigM \; ArtBudget$$

Secondary objective: Maximize the total discounted customer satisfaction score generated by the selected projects.

$$ MAX \; Score = \sum_{p \in P} \sum_{t = EF_p}^{LF_p} \; \frac{s_p\;z_{p,t}}{(1 + \alpha)^t} - BigM \; ArtBudget$$

Remark: Notice that if the artificial variable $ArtBudget$ is positive, we penalize it with a large value coefficient. The idea is that the artificial variable will be positive only to make the problem feasible.

Subject to the following constraints:

A project is selected if it is in the set $IN$ $$ y_p = 1 \;\;\;\; \forall p \in IN $$

A project is not selected if it is in the set $OUT$ $$ y_p = 0 \;\;\;\; \forall p \in OUT $$

This constraint ensures that a selected project starts at some period within the starting time window, i.e. in the interval $[ES(p), \; LS(p)]$. $$ \sum_{t = ES(p)}^{LS(p)} x_{p, t} = y_{p} \;\;\;\;\forall p \in P $$

This constraint ensures that a selected project starts at the release date or after. $$ \sum_{t = ES(p)}^{LS(p)} t \; x_{p, t} \geq \lambda_{p} \; y_{p} \;\;\;\; \forall p \in P $$

This constraint ensures that a selected project starts at the release date or before. $$ \sum_{t = ES(p)}^{LS(p)} (t + d_{p}) \; x_{p, t} \leq \delta_{p} \; y_{p} \;\;\;\; \forall p \in P $$

This constraint ensures that a selected project is completed at some period within the finishing time window, i.e. in the interval $[EF(p), \; LF(p)]$ $$ \sum_{t = EF(p)}^{LF(p)} z_{p, t} = y_{p} \;\;\;\; \forall p \in P $$

This constraint ensures that a project that starts at a time period $t$, then this project is completed at a time period equal to the time period it started plus the duration of the project, i.e. $t + d_p$ $$ \sum_{t = EF(p)}^{LF(p)} t \; z_{p, t} = \sum_{t = ES(p)}^{LS(p)} (t + d_{p}) \; x_{p, t} \;\;\;\; \forall p \in P $$

This constraint ensures that the cost of the selected projects in the portfolio does not exceed the budget available. In case the cost of the selected projects is larger than the budget, then the value of the artificial variable $ArtBudget$ determines the additional budget required to fund the cost of the selected projects. $$ \sum_{p \in P} c_{p} \; y_{p} \leq \beta + ArtBudget $$

Non-negativity and integrality constraints

The variables that select the projects as part of the optimal portfolio are binary. $$ y_p \in \{0,1\} \;\;\;\; \forall p \in P $$

The start and completion times of the selected projects are binary variables. $$ x_{p,t},\;z_{p,t} \in \{0,1\} \;\;\;\; \forall p \in P \;\; \forall t \in T $$

The artificial variable that determines if an extra budget is required is a non-negative continuous variable. $$ ArtBudtet \geq 0 $$

Script Output and KPIs¶

When the optimization is completed, a simple report showing the list of selected projects and its start period is displayed. Also, a variety of KPIs are computed to provide helpful information about the selected portfolio.

$Projects\;selected = \sum_{p \in P} y_p$: number of projects selected.

$Project \; selection \; performance = \frac{\sum_{p \in P} y_p}{|P|} $: ratio of selected projects respect to all projects in the portfolio.

$Total\;cost = \sum_{p \in P} c_p \; y_p$: cost of selected projects.

$Total\;benefit = \sum_{p \in P} v_p \; y_p$: total benefit of selected projects.

$Cost\;by\;period$: this KPI assumes that the project cost $c_p$ is uniformly distributed during the project duration $d_p$.

$Benefit\;by\;period$: this KPI assumes that the project benefit $v_p$ occurs at project completion.

$Benefit\;performance = \frac{Total\;benefit}{\sum_{p \in P} v_p}$: ratio of selected benefit and possible benefit from all projects in portfolio.

$Budget\;utilization = \frac{Total\;cost}{\beta}$: ratio of total cost and available budget for projects.

$Cost\;performance = \frac{Total\;cost}{\sum_{p \in P} c_p}$: ratio of selected cost and possible total cost.

$ROI = \frac{Total\;benefit}{Total\;cost}$: ROI of selected portfolio.

$Available\;budget = \beta$: Available budget for the optimization.

$Extra\;budget\;required = ArtBudget$: extra budget required to execute selected portfolio.

Remarks¶

Each project belongs to an investment area. KPIs can be calculated at the investment area level to understand the impact of the optimal portfolio on the investment areas.

Moreover, constraints can be added to the MIP model formulation to reflect portfolio management strategy related to the investment areas. For example, for an IT organization, portfolio executives want to ensure that at least 30% of the total cost of the optimal portfolio is invested on IT projects of the R&D investment area.

# The setObjectiveN() method of the model object m allows to define multiple objectives.
# The first argument is the linear expression defining the most important objective, called primary objective, in this case 
# it is the minimization of extra workers required to satisfy shift requirements. The second argument is the index of the 
# objective function, we set the index of the primary objective to be equal to 0. The third argument is the priority of the 
# objective. The fourth argument is the relative tolerance to degrade this objective when a lower priority
# objective is optimized. The fifth argument is the name of this objective.
# A hierarchical or lexicographic approach assigns a priority to each objective, and optimizes for the objectives in 
# decreasing priority order. For this problem, we have two objectives, and this objective has the highest priority which is
# equal to 2. When the secondary objective is minimized, since the relative tolerance is 0.2, we can only degrade the 
# minimum number of extra workers up to 20%. 

# Setting first objective with an absolute tolerace of 5% and relative tolerance of 0
ipo_model.setObjectiveN(objective1, index = 0, priority = 2, abstol = 2.0,
                        reltol = 0.2, name = "Project_Benefit")


# Setting second objective 
ipo_model.setObjectiveN(objective2, index = 1, priority = 1, name = "customer_satisfaction")

                benefit_by_period[t + project_duration[project]] = \
                    benefit_by_period[t + project_duration[project]] + project_benefit[project]
                break
        total_cost = total_cost + project_cost[project]
        total_benefit = total_benefit + project_benefit[project]
        projects_selected = projects_selected + 1
